Clae (pronounced "Clay") will automatically use the Cover and Logo you upload in your Ghost settings. For best results, use a darker, wide, panorama like cover, and a light, if not white logo with a transparent background. Note that the logo will be scaled to match the header text, but the background will be trimmed to cover the area, usually losing the left and right of the image, as it is centered vertically. This will vary according to screen size, and as such your header image should be at least 1920x1080px, and a .jpg to save on file size. JPEGMini is a good site for unnoticeable compression.

The theme is both responsive and touch-ready, and incorporates the latest web technologies, and the most graceful of fallback options, providing a near seamless experience from Internet Explorer 7 through to the latest and greatest browsers.

We wish to express thanks to Flat UI Colors for their wonderful shades of blue, and FlatIcons.net for their royalty free up arrow. It is, almost always, the little things.

Please note that neither the cover nor the logo are included in this package, and that the theme name in the footer changes with the site title to match the header, and is not required credit nor attribution.

Installing this Theme - top

Partly quoted from the Ghost Documentation.

Download the theme package of your choice, extract it and place it in content/themes. It should be in a folder called clae alongside another called casper.

To switch to your newly purchased theme:

  1. Restart Ghost. At the moment, Ghost won't notice that you've added a new folder to content/themes so you'll need to restart it
  2. Login to your Ghost admin, and navigate to /ghost/settings/general/
  3. Select "Clae" in the Theme options dropdown
  4. Click 'Save'
  5. Visit the frontend of your blog and marvel at your new theme.

Handlebars - top

Handlebars is the framework that Ghost uses for theming. It is why the theme files are called index.hbs, post.hbs and default.hbs. Because this theme conforms to the Ghost guidelines, the theme is split into these three files.

post.hbs is called when an individual post is being displayed.

index.hbs is called when the homepage is being called, and lists the most recent posts and handles pagination.

default.hbs is called from both index.hbs and post.hbs and contains the header, footer, and other, behind the scenes codes that appears n every page.

All of these files contain the general structure of the site, and handle only a minimal amount of logic. You should probably leave them alone.

CSS - top

The CSS comes in both minified and standard versions, stored in the css folder.

The file style.min.css, and it's father, style.css, contain the actual theme styles.

If you are gong to change anything, I would recommend switching from style.min.css to style.css in default.hbs.

A rudimentary understanding of CSS is required to modify the theme, for instance, to change the header text from white to black I would make the following change:

header h1 a { color: #ffffff; } → header h1 a { color: #000000; }

To switch the footer text to black to match, I would need to make three changes:

footer h1 a { color: #eeeeee; } → footer h1 a { color: #000000; }

footer a { color: #eeeeee; } → footer a { color: #000000; }

footer { color: #cccccc; } → footer { color: #000000; }

You can use a HEX color chooser to find less harsh colors to your liking.

JavaScript - top

JavaScript is kept to a minimum, and is only to avoid degradation in older browsers and smooth the scroll-to-top motion. jQuery is included by Ghost itself, and no external assets are loaded by Clae. At the bottom of default.hbs there is a small inline snippet so when users click the arrow in the footer, the page is smoothly scrolled to the top.

Restrictions - top

This theme itself is about as unlimited as can currently be made for Ghost. Unfortunately, Ghost, at this time, does not support comments or browsing by tags. Nor does it support Author pages. It does, however, deliver what it promised to deliver.

Good, clean, simple blogging.

This theme was originally published on ThemeForest, but has since been removed and is now open-source, although entirely unsupported.
